Subversion

CPF and CPF plug-ins use the Apache Subversion source code control system to track changes to the source code and tag versions for each release. Developers must have an understanding of using Subversion for source code control.

Subversion command line and GUI tools can be downloaded from the following sites.

  • http://subversion.apache.org/">Subversion - Command line tools
  • Eclipse Subversion Team Provider
  • Tortoise SVN Windows GUI

See the Version Control with Subversion

for details on using Subversion.

The CPF source code can be accessed using the following Subversion repository within the BC Government network or VPN. Plug-in developers do not need the source code to develop a plug-in but it is available for reference purposes.

  • http://apps.bcgov/svn/cpf/api-source/trunk/ - CPF Subversion repository for source code download
  • http://apps.bcgov/wsvn/TC7.cpf-api - CPF Subversion web interface to browse source code

Developers will need to deliver their plug-ins via a subversion repository. A request should be sent their Business Analyst that a subversion repository and user account be created for their project.
